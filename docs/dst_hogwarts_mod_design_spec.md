## 1. 设计目标与边界

### 1.1 核心体验

> 让玩家在《饥荒联机版》的生存框架中体验霍格沃茨魔法学院的学习、成长与冒险，以**休闲娱乐为主、兼顾生存挑战**。

- 沉浸式魔法学习与施法体验
- 丰富的可放置建筑与手持物品扩展
- 任务驱动的中期核心循环
- 多人协作（合作酿药、团队魁地奇）

### 1.2 非目标

- ❌ 不修改引擎底层逻辑
- ❌ 不修改原版核心战斗系统
- ❌ 不做世界生成内容（v0.1 不含自定义地形/遗迹/传送门）
- ❌ 不追求电竞级 PVP 平衡
- ❌ 不做单机版（DS）兼容

### 1.3 约束

- 所有内容通过 `modmain.lua` 与 DST 公开 API 实现
- 不依赖其他第三方模组
- 联机以**主机权威**为基础架构
- 美术资源使用饥荒原版 Spriter 风格

## 3. 统一设计规范

### 3.1 命名规范

| 类别 | 规则 | 示例 |
|------|------|------|
| Prefab 名 | `hw_` 前缀 + 蛇形命名 | `hw_magic_workbench`, `hw_wand_elder` |
| Strings 键 | `STRINGS.HW.<CATEGORY>.<NAME>` | `STRINGS.HW.ITEMS.WAND_ELDER` |
| Asset 文件 | `hw_<type>_<name>.zip` | `hw_anim_wand_elder.zip` |
| 标签（Tag） | `hw_<tag>` | `hw_wizard`, `hw_herbology` |
| 组件（Component） | `hw_<component>` | `hw_mana`, `hw_spellcaster` |

### 3.2 平衡与数值标尺

**稀有度体系**

| 稀有度 | 颜色 | 获取难度 | 示例 |
|--------|------|----------|------|
| 普通（Common） | 白 | 基础采集 | 普通草药种子 |
| 精良（Uncommon） | 绿 | 简单制作 | 初级魔杖 |
| 稀有（Rare） | 蓝 | 任务/复合制作 | 高级魔药 |
| 史诗（Epic） | 紫 | Boss/长线任务 | 接骨木魔杖、阿瓦达索命 |

**核心数值锚点**

- 法力值上限基础：**100**（可通过成长提升至 200）
- 法力恢复速率：**1/s**（站立）、**2/s**（靠近书架）、**0.3/s**（战斗中）
- 施法理智副作用：每次施法消耗 **3~15 理智**（按法术等级递增）
- 魔杖耐久基础：**100 次施法**（可修复）
- 冷却基准：普通法术 **5s**、强力法术 **15~30s**、终极法术 **60s+**

### 3.3 联机规则

- **主机权威**：所有数值计算、状态转移在服务端执行
- **客户端预测**：施法动画、移动预测在客户端播放，服务端校验后回滚
- **可同步字段**：法力值、冷却计时器、任务进度、魔杖耐久——通过 `net_*` 变量同步
- **RPC**：施法请求、任务交互通过 `SendModRPCToServer` / `SendModRPCToClient`

### 3.4 兼容性

- 所有内容在 `modmain.lua` 中注册，不覆盖原版 prefab
- 通过 `AddPrefabPostInit` 钩子扩展原版实体（如给玩家添加 `hw_mana` 组件）
- 已知冲突点：其他法术模组若使用相同按键绑定需提供配置项
- Mod 配置菜单（`modinfo.lua`）暴露关键数值开关供服务器管理员调整

#### 4.A.1 魔法工作台（Magic Workbench）

**概念与定位**

模组的核心制作中心，解锁所有 Hogwarts 物品的制作配方。属于**早期**必建建筑，类似原版炼金引擎。

**获取方式**

- 制作配方：金块 ×4 + 石砖 ×6 + 噩梦燃料 ×2
- 解锁条件：无（模组加载后即可在制作栏"魔法"标签页中看到）

**交互流程**

1. 玩家靠近工作台 → 制作栏出现"霍格沃茨"制作页签
2. 选择配方 → 检查背包材料 → 制作成功 → 物品进入背包
3. 部分高级配方需要额外条件（如持有魔杖、完成特定任务）

**数值与公式**

| 属性 | 值 |
|------|----|
| 生命值 | 400 |
| 制作范围 | 4 Tiles |
| 可被锤子拆除 | 是（返还 50% 材料） |

**联机与同步**

- 制作操作由服务端验证，客户端仅显示 UI
- 无需额外同步字段

**美术与音效需求**

- 📦 放置动画（idle / place / hit / broken）
- 🔊 放置音效、制作成功音效（类似魔法闪烁声）

**实现要点（Coding AI 用）**

- Prefab：`hw_magic_workbench`
- 组件：`placer`, `workable`, `lootdropper`
- 通过 `GLOBAL.RECIPETABS` 新增"霍格沃茨"制作页签
- 通过 `AddRecipe2` 注册所有配方，`tech_level` 指向本工作台

**验收标准**

- [ ] 可制作、放置、拆除
- [ ] 靠近后制作栏出现专属页签
- [ ] 高级配方在条件不满足时灰显
- [ ] 联机环境下多人可同时使用

#### 4.A.2 书架（Bookshelf）

**概念与定位**

知识与法术学习的载体。玩家将"魔法书"放到书架上，靠近书架阅读可学习/升级法术。属于**早中期**建筑，同时提供法力恢复加速光环。

**获取方式**

- 制作配方：木板 ×4 + 莎草纸 ×2 + 噩梦燃料 ×1
- 需在魔法工作台旁制作

**交互流程**

1. 放置书架 → 右键打开书架界面（3 格容器）
2. 将"魔法书"物品放入书架 → 书架显示已存书籍
3. 玩家靠近书架按"阅读"键 → 播放阅读动画（3s）→ 学会对应法术 / 提升法术熟练度
4. 每本书可被阅读有限次（耐久递减），耗尽后书籍消失

**数值与公式**

| 属性 | 值 |
|------|----|
| 生命值 | 200 |
| 容量 | 3 格 |
| 法力恢复光环范围 | 6 Tiles |
| 法力恢复加成 | +1/s（叠加到基础恢复上） |
| 阅读时间 | 3s |

**联机与同步**

- 容器内容服务端权威，客户端通过 `net_string` 同步显示
- 法力恢复光环通过 `AreaAware` 组件在服务端计算

**美术与音效需求**

- 📦 idle（空/有书两状态）、place、hit、broken
- 🔊 翻书音效、学习成功魔法音效

**实现要点（Coding AI 用）**

- Prefab：`hw_bookshelf`
- 组件：`container`（3 格，只接受 `hw_tag_spellbook`），`workable`，`hw_mana_aura`（自定义光环组件）
- 阅读交互：`ACTIONS.HW_READ`，服务端回调检查背包、扣耐久、调用 `hw_spellcaster:LearnSpell()`

**验收标准**

- [ ] 可放置、拆除，只接受魔法书物品
- [ ] 阅读后正确学会法术或提升熟练度
- [ ] 法力恢复光环范围与加成数值正确
- [ ] 联机环境下多人可同时阅读不同书籍

#### 4.A.3 草药种植园（Herbology Garden）

**概念与定位**

种植魔法草药的专用设施，是魔药体系的资源入口。属于**中期**建筑。

**获取方式**

- 制作配方：木板 ×2 + 便便 ×4 + 石头 ×2
- 需在魔法工作台旁制作

**交互流程**

1. 放置种植园（占 2×2 地面）
2. 将草药种子放入（右键）→ 开始生长
3. 共 3 个生长阶段：幼苗 → 成长 → 成熟（可收获）
4. 收获后返回幼苗阶段，可持续循环
5. 需定期施肥（便便/腐烂物），否则生长停滞

**数值与公式**

| 草药 | 生长周期（天） | 每次收获量 | 稀有度 |
|------|----------------|------------|--------|
| 曼德拉草根 | 4 | 1 | 稀有 |
| 毒触手汁液草 | 3 | 2 | 精良 |
| 水仙根粉草 | 2 | 3 | 普通 |
| 月光花 | 5（仅夜间生长） | 1 | 稀有 |

- 施肥一次加速 1 天进度
- 冬季生长速度 ×0.5，春季 ×1.5

**联机与同步**

- 生长阶段 `net_byte` 同步，客户端切换贴图
- 收获操作服务端验证

**美术与音效需求**

- 📦 3 阶段生长贴图 × 4 种草药 + 空地状态
- 🔊 种植、施肥、收获音效

**实现要点（Coding AI 用）**

- Prefab：`hw_herb_garden`
- 组件：`hw_growable`（自定义生长组件，含阶段、施肥、季节修正）、`pickable`、`workable`
- 种子 Prefab：`hw_seed_mandrake`、`hw_seed_tentacle_juice`、`hw_seed_daffodil`、`hw_seed_moonflower`

**验收标准**

- [ ] 种植、施肥、收获流程完整
- [ ] 生长周期受季节修正影响
- [ ] 未施肥时生长停滞
- [ ] 联机环境下生长状态同步

#### 4.A.4 魁地奇球场（Quidditch Pitch）

**概念与定位**

娱乐与任务入口，提供"追逐金色飞贼"小游戏。属于**中期**大型建筑，需要多人协作建造。

**获取方式**

- 制作配方：木板 ×10 + 绳子 ×6 + 金块 ×4 + 红宝石 ×2
- 需在魔法工作台旁制作，占地 5×5 Tiles

**交互流程**

1. 放置球场后，右键交互 → 选择"开始比赛"
2. 需至少 2 名玩家在球场范围内
3. 比赛开始 → 金色飞贼生成并在球场范围内高速飞行
4. 骑飞天扫帚的玩家可追逐并捕捉飞贼
5. 抓到飞贼 → 比赛结束 → 奖励分发
6. 限时 3 分钟，超时飞贼逃走，无奖励

**数值与公式**

| 属性 | 值 |
|------|----|
| 建筑生命值 | 800 |
| 比赛时长 | 180s |
| 飞贼速度 | 玩家骑扫帚速度 × 1.2 |
| 捕获奖励 | 金色飞贼物品 ×1 + 经验 + 随机魔法书 |
| 冷却（再次比赛） | 1 游戏日 |

**联机与同步**

- 比赛状态机（空闲/准备/进行中/结束）服务端权威
- 飞贼位置服务端计算，客户端插值平滑
- 捕获判定服务端碰撞检测

**美术与音效需求**

- 📦 球场框架、球门柱、飞贼动画
- 🔊 比赛开始号角、飞贼飞行声、捕获欢呼声

**实现要点（Coding AI 用）**

- Prefab：`hw_quidditch_pitch`、`hw_golden_snitch_game`（比赛中飞贼实体）
- 组件：`hw_minigame_host`（管理比赛状态机）、`hw_flyable_target`（飞贼 AI 移动）
- 通过 RPC 同步比赛事件

**验收标准**

- [ ] 2+ 人可启动比赛
- [ ] 飞贼 AI 移动流畅，不穿墙
- [ ] 捕获判定准确
- [ ] 冷却期内无法再次开启

#### 4.B.1 魔杖（Wand）

**概念与定位**

施法的核心载体，玩家必须装备魔杖才能施展魔法。魔杖有成长系统，长期使用同一根魔杖可提升属性。属于**早期**获取、贯穿全程的核心装备。

**获取方式**

| 魔杖 | 配方 | 稀有度 | 特性 |
|------|------|--------|------|
| 学徒魔杖 | 树枝 ×2 + 噩梦燃料 ×1 | 精良 | 基础，无加成 |
| 凤凰羽毛杖 | 活木 ×2 + 红宝石 ×1 + 羽毛 ×3 | 稀有 | 火系法术伤害 +20% |
| 接骨木魔杖 | 活木 ×4 + 噩梦燃料 ×4 + 紫宝石 ×2 | 史诗 | 全法术消耗 -20%，冷却 -15% |

**交互流程**

1. 装备到手持栏位
2. 按快捷键（默认 Z）打开法术轮盘 → 选择法术 → 选择目标 → 施法
3. 施法消耗法力 + 魔杖耐久 + 少量理智
4. 耐久为 0 时魔杖不可施法，可在魔法工作台修复

**魔杖成长系统**

- 每次施法获得 **1 点魔杖经验**
- 每 50 点经验升 1 级，最高 **5 级**
- 每级提升：耐久上限 +10%，法力消耗 -3%

**数值与公式**

| 属性 | 学徒 | 凤凰羽毛 | 接骨木 |
|------|------|----------|--------|
| 基础耐久（次） | 100 | 150 | 200 |
| 施法速度倍率 | ×1.0 | ×1.1 | ×1.2 |
| 修复成本 | 噩梦燃料 ×1 | 噩梦燃料 ×2 + 红宝石 ×1 | 噩梦燃料 ×3 + 紫宝石 ×1 |

**联机与同步**

- 魔杖耐久、等级、经验通过 `net_ushort` 同步
- 施法请求通过 RPC → 服务端验证后执行

**美术与音效需求**

- 📦 3 种魔杖的手持/地面贴图、挥舞动画
- 🔊 装备、挥舞、施法、损坏音效

**实现要点（Coding AI 用）**

- Prefab：`hw_wand_apprentice`、`hw_wand_phoenix`、`hw_wand_elder`
- 组件：`hw_wand`（耐久/经验/等级/特性）、`equippable`、`finiteuses`
- 法术轮盘 UI：客户端 widget，按 Z 呼出，选择后发送 RPC

**验收标准**

- [ ] 三种魔杖可制作、装备、施法
- [ ] 耐久正确消耗与修复
- [ ] 成长系统正确计算经验与等级加成
- [ ] 法术轮盘 UI 流畅

**获取方式**

| 草药 | 直接食用效果 | 保质期（天） | 堆叠上限 |

| 水仙根粉草 | +5 饥饿，+5 理智 | 8 | 20 |
| 月光花 | +30 理智，+10 法力 | 4 | 5 |

**联机与同步**

- 使用标准 `inventoryitem` 组件，无需额外同步

**美术与音效需求**

- 📦 4 种草药地面/背包图标

**验收标准**

#### 4.B.3 魔药（Potions）

**概念与定位**

消耗品，提供强力临时增益。通过草药 + 其他材料在魔法工作台炼制。属于**中后期**核心消耗品。

**魔药列表**

| 魔药 | 配方 | 效果 | 持续时间 | 冷却 |
|------|------|------|----------|------|
| 治疗药水 | 曼德拉草根 ×2 + 水仙根粉草 ×1 | 5s 内回复 60 生命 | 5s | 30s |
| 法力药水 | 月光花 ×2 + 水仙根粉草 ×1 | 立即恢复 50 法力 | 即时 | 20s |
| 隐身药水 | 月光花 ×1 + 毒触手汁液草 ×2 | 60s 内对怪物隐身 | 60s | 120s |
| 力量药水 | 曼德拉草根 ×1 + 毒触手汁液草 ×1 + 怪物肉 ×1 | 伤害 +50% | 90s | 180s |
| 福灵剂 | 月光花 ×2 + 曼德拉草根 ×2 + 金块 ×1 | 180s 内掉落率翻倍、暴击率 +25% | 180s | 1 游戏日 |

**联机与同步**

- 增益 buff 通过 `net_bool` 同步状态，服务端计时

**美术与音效需求**

- 📦 5 种药瓶图标与饮用动画
- 🔊 喝药音效、增益激活音效

**实现要点（Coding AI 用）**

- Prefab：`hw_potion_heal`、`hw_potion_mana`、`hw_potion_invis`、`hw_potion_strength`、`hw_potion_felix`

## 6. 研发拆分（给 Coding AI）

### 6.1 里程碑

| 里程碑 | 内容 | 预计工作量 |
|--------|------|------------|
| M1 — 基础框架 | modmain 框架、hw_mana 组件、魔法工作台、制作页签 | 3~5 天 |
| M2 — 核心物品 | 3 种魔杖、书架、法术轮盘 UI | 5~7 天 |
| M3 — 魔法系统 | 6 个法术实现、Buff 系统、投射物 | 7~10 天 |
| M4 — 资源循环 | 草药种植园、4 种草药、5 种魔药、飞天扫帚 | 5~7 天 |
| M5 — 任务与魁地奇 | 任务系统、全部任务、魁地奇球场、金色飞贼 | 7~10 天 |
| M6 — 联机测试与打磨 | 联机同步调优、PVP 平衡、性能优化、Bug 修复 | 5~7 天 |

### 6.2 风险点

1. **联机同步延迟** — 法术投射物与碰撞判定在高延迟下可能不准确 → 服务端做宽松判定 + 客户端插值
2. **魁地奇飞贼 AI** — 高速移动 AI 可能卡地形 → 限制在球场范围、添加边界碰撞
3. **Buff 叠加冲突** — 多个 Buff 同时生效可能产生意外交互 → `hw_buff_manager` 需实现优先级与互斥规则
4. **Mod 兼容性** — 其他添加 `RECIPETABS` 的模组可能冲突 → 使用独特命名空间
5. **性能** — 火焰风暴 AOE 检测大量实体 → 使用空间分区、限制最大命中数
