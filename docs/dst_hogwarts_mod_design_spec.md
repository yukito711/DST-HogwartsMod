## 1. 设计目标与边界

### 1.1 核心体验

> 让玩家在《饥荒联机版》的生存框架中体验霍格沃茨魔法学院的学习、成长与冒险，以**休闲娱乐为主、兼顾生存挑战**。

- 沉浸式魔法学习与施法体验
- 丰富的可放置建筑与手持物品扩展
- 任务驱动的中期核心循环
- 多人协作（合作酿药等）

### 1.2 非目标

- ❌ 不修改引擎底层逻辑
- ❌ 不修改原版核心战斗系统
- ❌ 不做世界生成内容（v0.1 不含自定义地形/遗迹/传送门）
- ❌ 不追求电竞级 PVP 平衡
- ❌ 不做单机版（DS）兼容

### 1.3 约束

- 所有内容通过 `modmain.lua` 与 DST 公开 API 实现
- 不依赖其他第三方模组
- 联机以**主机权威**为基础架构
- 美术资源使用饥荒原版 Spriter 风格

## 3. 统一设计规范

### 3.1 命名规范

| 类别 | 规则 | 示例 |
|------|------|------|
| Prefab 名 | `hw_` 前缀 + 蛇形命名 | `hw_magic_workbench`, `hw_wand` |
| Strings 键 | `STRINGS.HW.<CATEGORY>.<NAME>` | `STRINGS.HW.ITEMS.WAND` |
| Asset 文件 | `hw_<type>_<name>.zip` | `hw_anim_wand.zip` |
| 标签（Tag） | `hw_<tag>` | `hw_wizard`, `hw_herbology` |
| 组件（Component） | `hw_<component>` | `hw_mana`, `hw_spellcaster` |

### 3.2 平衡与数值标尺

**核心数值锚点**

- 法力值上限基础：**100**（可通过成长提升至 200）
- 法力恢复速率：**1/s**（站立）、**0.3/s**（战斗中）
- 理智门槛（不消耗）：施法不扣除理智，但需当前理智达到门槛。生活/工具类 > 50；攻击/治疗类 > 50（效果随理智衰减）；不可饶恕咒 > 200
- 冷却基准：普通法术 **5s**、强力法术 **15~30s**、终极法术 **60s+**

### 3.3 联机规则

- **主机权威**：所有数值计算、状态转移在服务端执行
- **客户端预测**：施法动画、移动预测在客户端播放，服务端校验后回滚
- **可同步字段**：法力值、冷却计时器、任务进度、魔杖契合度——通过 `net_*` 变量同步
- **RPC**：施法请求、任务交互通过 `SendModRPCToServer` / `SendModRPCToClient`

### 3.4 兼容性

- 所有内容在 `modmain.lua` 中注册，不覆盖原版 prefab
- 通过 `AddPrefabPostInit` 钩子扩展原版实体（如给玩家添加 `hw_mana` 组件）
- 已知冲突点：其他法术模组若使用相同按键绑定需提供配置项
- Mod 配置菜单（`modinfo.lua`）暴露关键数值开关供服务器管理员调整

#### 4.A.1 魔法工作台（Magic Workbench）

**概念与定位**

模组的核心制作中心，解锁所有 Hogwarts 物品的制作配方。属于**早期**必建建筑，类似原版炼金引擎。

**获取方式**

- 制作配方：金块 ×4 + 石砖 ×6 + 噩梦燃料 ×2
- 解锁条件：无（模组加载后即可在制作栏"魔法"标签页中看到）

**交互流程**

1. 玩家靠近工作台 → 制作栏出现"霍格沃茨"制作页签
2. 选择配方 → 检查背包材料 → 制作成功 → 物品进入背包
3. 部分高级配方需要额外条件（如持有魔杖、完成特定任务）

**数值与公式**

| 属性 | 值 |
|------|------|
| 生命值 | 400 |
| 制作范围 | 4 Tiles |
| 可被锤子拆除 | 是（返还 50% 材料） |

**联机与同步**

- 制作操作由服务端验证，客户端仅显示 UI
- 无需额外同步字段

**美术与音效需求**

- 📦 放置动画（idle / place / hit / broken）
- 🔊 放置音效、制作成功音效（类似魔法闪烁声）

**实现要点（Coding AI 用）**

- Prefab：`hw_magic_workbench`
- 组件：`placer`, `workable`, `lootdropper`
- 通过 `GLOBAL.RECIPETABS` 新增"霍格沃茨"制作页签
- 通过 `AddRecipe2` 注册所有配方，`tech_level` 指向本工作台

**验收标准**

- [ ] 可制作、放置、拆除魔法工作台
- [ ] 靠近后制作栏出现专属页签
- [ ] 制作配方可以正常使用
- [ ] 高级配方在条件不满足时灰显
- [ ] 联机环境下多人可同时使用

#### 4.A.2 书架（Bookshelf）

**概念与定位**

法术学习与升级的制作站。功能与魔法工作台类似——玩家靠近书架后，制作栏出现"霍格沃茨"制作页签，可在其中**学习新法术**和**升级已有法术**。属于**早中期**建筑。

**获取方式**

- 制作配方：木板 ×4 + 莎草纸 ×2 + 噩梦燃料 ×1
- 需在魔法工作台旁制作

**交互流程**

1. 放置书架 → 玩家靠近 → 制作栏出现"学习魔法"制作页签
2. 页签中显示可学习 / 升级的法术配方
3. 法术配方分为**初级、中级、高级**三个等级（详见 4.C 法术等级体系）
4. 选择配方 → 检查材料与前置条件 → 制作成功 → 学会法术 / 法术等级提升

**数值与公式**

| 属性 | 值 |
|------|------|
| 生命值 | 200 |
| 制作范围 | 4 Tiles |
| 可被锤子拆除 | 是（返还 50% 材料） |

**联机与同步**

- 制作操作由服务端验证，客户端仅显示 UI
- 无需额外同步字段（与魔法工作台一致）

**美术与音效需求**

- 📦 idle、place、hit、broken
- 🔊 放置音效、法术学习成功音效（魔法闪烁声）

**实现要点（Coding AI 用）**

- Prefab：`hw_bookshelf`
- 组件：`placer`、`workable`、`lootdropper`
- 与魔法工作台共享同一个 `RECIPETABS` 制作页签，或新增专属"法术学习"子页签
- 通过 `AddRecipe2` 注册法术学习 / 升级配方，`tech_level` 指向本书架
- 中级法术配方的前置条件通过 `AddRecipe2` 的自定义 `condition` 回调验证玩家是否已学会对应初级法术（`hw_spellcaster:HasSpell(spell_id, 1)`），高级同理

**验收标准**

- [ ] 可制作、放置、拆除书架
- [ ] 靠近后制作栏出现专属页签，显示法术配方
- [ ] 法术配方按初级 / 中级 / 高级正确显示

- [ ] 联机环境下多人可同时使用

#### 4.A.3 草药种植园（Herbology Garden）

**概念与定位**

种植魔法草药的专用设施，是魔药体系的资源入口。属于**中期**建筑。

**获取方式**

- 制作配方：木板 ×2 + 便便 ×4 + 石头 ×2 + 噩梦燃料 ×2
- 需在魔法工作台旁制作

**交互流程**

1. 放置种植园（占 2×2 地面）
2. 将草药种子放入（右键）→ 开始生长
3. 共 3 个生长阶段：幼苗 → 成长 → 成熟（可收获）
4. 收获后返回幼苗阶段，可持续循环
5. 需定期施肥（便便/腐烂物），否则生长停滞

**数值与公式**

| 草药 | 生长周期（天） | 每次收获量 |
|------|----------------|------------|
| 白鲜（Dittany） | 3 | 3 |
| 苦艾草（Wormwood） | 3 | 3 |
| 缬草（Valerian） | 3 | 3 |

- 冬季生长速度 ×0.5，春季 ×1.5

**联机与同步**

- 生长阶段 `net_byte` 同步，客户端切换贴图
- 收获操作服务端验证

**美术与音效需求**

- 📦 3 阶段生长贴图 × 3 种草药 + 空地状态
- 🔊 种植、施肥、收获音效

**实现要点（Coding AI 用）**

- Prefab：`hw_herb_garden`
- 组件：`hw_growable`（自定义生长组件，含阶段、施肥、季节修正）、`pickable`、`workable`
- 种子 Prefab：`hw_seed_dittany`、`hw_seed_wormwood`、`hw_seed_valerian`

**验收标准**

- [ ] 种植、施肥、收获流程完整
- [ ] 生长周期受季节修正影响
- [ ] 未施肥时生长停滞
- [ ] 联机环境下生长状态同步

#### 4.B.1 魔杖（Wand）

**概念与定位**

施法的核心载体，玩家必须装备魔杖才能施展魔法。每根魔杖在首次装备时与巫师绑定，生成一个随机**契合度**数值，影响攻击与治疗类法术的效果——体现"魔杖选择巫师"的设定。属于**早期**获取、贯穿全程的核心装备。

**获取方式**

- 制作配方：活木 ×4 + 噩梦燃料 ×2
- 需在魔法工作台旁制作

**交互流程**

1. 装备到手持栏位
2. 按快捷键（默认 Z）打开法术轮盘 → 选择法术 → 选择目标 → 施法
3. 施法消耗法力（需当前理智达到法术门槛值，**不消耗理智**，魔杖无耐久消耗）
4. 首次装备时触发"绑定"特效，生成契合度并显示给玩家

**魔杖契合度系统（Wand Affinity）**

- 玩家首次装备魔杖时，系统为该玩家-魔杖组合随机生成一个**契合度**值（范围 0.5 ~ 1.5）
- 契合度永久绑定，不可更改；想换契合度需重新制作魔杖
- 契合度作为**乘数**参与攻击/治疗类法术的最终效果计算：`最终效果 = 基础值 × 理智衰减倍率 × 契合度`
- 生活/工具类与不可饶恕咒**不受**契合度影响
- 同一根魔杖被不同玩家装备时，各自拥有独立的契合度

**数值与公式**

| 属性 | 值 |
|------|------|
| 契合度范围 | 0.5 ~ 1.5（首次装备时随机生成） |
| 契合度影响 | 攻击/治疗类法术最终效果 × 契合度 |
| 耐久 | 无（魔杖不消耗耐久） |
| 绑定规则 | 每位玩家对每根魔杖拥有独立契合度，首次装备时确定 |

**联机与同步**

- 魔杖契合度通过 `net_float` 同步
- 施法请求通过 RPC → 服务端验证后执行

**美术与音效需求**

- 📦 魔杖手持/地面贴图、挥舞动画
- 🔊 装备、挥舞、施法音效、绑定（首次装备）特效音

**实现要点（Coding AI 用）**

- Prefab：`hw_wand`
- 组件：`hw_wand`（契合度/绑定者映射表）、`equippable`
- 契合度存储：`wand.affinity_map[player_userid] = value`，保存在 `OnSave`/`OnLoad` 中
- 首次装备回调：`OnEquip` 中检查 `affinity_map`，未绑定则 `math.random() * 1.0 + 0.5` 生成契合度并保存
- 法术轮盘 UI：客户端 widget，按 Z 呼出，选择后发送 RPC

**验收标准**

- [ ] 魔杖可制作、装备、施法
- [ ] 首次装备时正确生成契合度并显示
- [ ] 不同玩家装备同一魔杖获得独立契合度
- [ ] 契合度正确参与攻击/治疗类法术效果计算
- [ ] 生活/工具类与不可饶恕咒不受契合度影响
- [ ] 法术轮盘 UI 流畅

#### 4.B.2 采摘草药（Harvested Herbs）

**概念与定位**

| 草药 | 直接食用效果 | 保质期（天） | 堆叠上限 |

| 缬草 | +15 理智，+5 饥饿 | 8 | 20 |

**联机与同步**

- 使用标准 `inventoryitem` 组件，无需额外同步

**美术与音效需求**

- 📦 3 种草药地面/背包图标
- 🔊 采摘、食用音效

**实现要点（Coding AI 用）**

- Prefab：`hw_herb_dittany`、`hw_herb_wormwood`、`hw_herb_valerian`

**验收标准**

#### 4.B.3 魔药（Potions）

**概念与定位**

消耗品，提供强力临时增益。通过原版**烹饪锅**炼制（制作时间与肉丸子一致，约 15s）。魔药可堆叠，**30 瓶一组**。属于**中后期**核心消耗品。

**魔药列表**

| 魔药 | 配方 | 效果 | 持续时间 | 冷却 |
|------|------|------|----------|------|
| 白鲜精华 | 白鲜 ×3 | 立即恢复 120 生命 | 即时 | 无 |
| 活地狱汤剂 | 白鲜 ×1 + 缬草 ×1 + 苦艾草 ×1 | 恢复 30 生命 + 恢复 50 法力 + 魔法效果 +10% | 600s | 600s |
| 福灵剂 | 鸟蛋 ×1 + 辣椒 ×1 + 苦艾草 ×1 + 缬草 ×1 | 180s 内掉落率翻倍、暴击率 +25% | 600s | 600s |

**联机与同步**

- 增益 buff 通过 `net_bool` 同步状态，服务端计时

**美术与音效需求**

- 📦 3 种药瓶图标与饮用动画
- 🔊 喝药音效、增益激活音效

**实现要点（Coding AI 用）**

- Prefab：`hw_potion_dittany_essence`、`hw_potion_living_hell`、`hw_potion_felix`
- 组件：`hw_potion`（管理 buff 类型、持续时间、冷却）、`stackable`（上限 30）
- 通过 `AddCookerRecipe("cookpot", ...)` 注册烹饪锅配方，`cooktime` 与肉丸子一致（0.75，即 15s）
- 配方优先级需高于原版食物，避免草药被意外煮成普通菜肴
- Buff 系统：`hw_buff_manager` 组件挂在玩家身上，管理所有活跃 buff

**验收标准**

- [ ] 三种魔药可制作、使用
- [ ] 白鲜精华立即恢复 120 生命
- [ ] 活地狱汤剂恢复 30 生命 + 50 法力，魔法效果 +10% 持续 600s
- [ ] 福灵剂效果与持续时间正确
- [ ] 冷却期内无法再次使用同类药水
- [ ] 联机环境下 buff 状态正确同步

#### 4.B.4 飞天扫帚（Flying Broom）

**概念与定位**

移动工具，提供高速飞行能力。属于**中期**获取。

**获取方式**

- 制作配方：活木 ×3 + 绳子 ×2 + 羽毛 ×4
- 需在魔法工作台旁制作

**交互流程**

1. 飞天扫帚放在背包中，**右键点击**即可骑乘（不占用手持栏，玩家可同时装备魔杖）
2. 骑乘状态：移速 ×3.0，**可攻击、可采摘、可施法**
3. 下马方式（以下任一条件触发）：再次右键点击扫帚；骑乘中被击中 **3 次**自动掉落；耐久为 0；扫帚被丢弃（从物品栏拖出）
4. 骑乘消耗耐久，每秒 -0.5 耐久

**数值与公式**

| 属性 | 值 |
|------|------|
| 移速倍率 | ×3.0 |
| 基础耐久 | 1800s（30分钟飞行时间） |
| 修复 | 活木 ×1 → +60s 耐久 |
| 骑乘时可施法 | 是 |
| 骑乘时可攻击 | 是 |
| 骑乘时可采摘 | 是 |
| 被击中掉落次数 | 3 次 |

**联机与同步**

- 骑乘状态 `net_bool`，耐久 `net_ushort`，受击计数 `net_byte`
- 移速修改通过 `locomotor` 组件，服务端权威

**美术与音效需求**

- 📦 骑乘、地面贴图 + 骑乘动画覆盖
- 🔊 起飞、飞行循环、降落音效

**实现要点（Coding AI 用）**

- Prefab：`hw_flying_broom`

### 4.C 魔法系统

---

**法力系统总览**

玩家在模组加载时获得 `hw_mana` 组件：

| 属性 | 基础值 | 说明 |
|------|--------|------|
| 法力上限 | 100 | 可通过成长提升至 200 |
| 自然恢复 | 1/s | 站立状态 |
| 战斗恢复 | 0.3/s | 近 5s 内有战斗行为时 |

**施法通用流程**

1. 玩家装备魔杖 → 按 Z 打开法术轮盘
2. 选择法术 → 选择目标（单体/地面/物体/自身）
3. 客户端播放预读动画 → 发送 RPC 到服务端
4. 服务端验证：法力是否足够？**当前理智是否 ≥ 门槛？** 冷却是否完成？目标是否合法？
5. 验证通过 → 扣资源 → 执行法术效果 → 同步给所有客户端
6. 验证失败 → 回滚客户端动画，显示失败提示

**法术分类与理智规则**

法术按功能分为三类，对应不同的理智门槛与成长规则：

| 法术分类 | 包含法术 | 理智门槛（不消耗） | 是否分等级 | 理智影响效果 |
|----------|----------|---------------------|------------|--------------|
| 生活 / 工具 | 火焰熊熊（Incendio） | 当前 > 50 | 否（仅基础版） | 否 |
| 攻击 / 治疗 | 神锋无影、火焰风暴、愈合如初 | 当前 > 50 | **是**（初级 → 中级 → 高级） | **是**（理智越低效果越差） |
| 不可饶恕咒 | 阿瓦达索命、魂魄出窍 | 当前 > 200 | 否（仅基础版） | 否 |

**攻击 / 治疗类法术的理智效果衰减**

| 当前理智 | 效果倍率 | 说明 |
|----------|----------|------|
| ≥ 150 | ×1.0 | 满效果 |
| 100 – 149 | ×0.75 | 轻微衰减 |
| 50 – 99 | ×0.5 | 半效 |
| ≤ 50 | — | 无法施法 |

> 🧠 **理智门槛系统**：施法**不消耗**理智值，但要求当前理智达到门槛才可释放。攻击与治疗类法术的伤害/治疗量还会随理智衰减——发疯时不仅无法使用强力法术，普通法术的效果也会打折。不可饶恕咒需要极高的精神集中力（> 200），只有理智充沛的巫师才能驾驭。

**法术等级体系（仅攻击 / 治疗类）**

攻击与治疗类法术（神锋无影、火焰风暴、愈合如初）分为 **初级（Basic）→ 中级（Intermediate）→ 高级（Advanced）** 三个等级，玩家通过在**书架**制作对应配方来学习与升级。其他法术（生活类、不可饶恕咒）无等级区分。

| 等级 | 解锁方式 | 配方前置条件 | 效果说明 |
|------|----------|--------------|----------|
| 初级 | 在书架制作初级法术配方 | 材料 | 基础版法术 |
| 中级 | 在书架制作中级法术配方 | 材料 + **已学会初级版** + **中级法术残页** | 数值增强（伤害 / 治疗 / 控制时长等提升） |
| 高级 | 在书架制作高级法术配方 | 材料 + **已学会中级版** + **高级法术残页** | 数值大幅增强，可能附加额外效果 |

> ✅ **技术可行性验证**：中 / 高级配方要求"已学会前一级法术"，通过 `AddRecipe2` 的 `condition` 回调实现（`hw_spellcaster:HasSpell(spell_id, level)`）。理智效果衰减通过施法时读取 `inst.components.sanity.current` 计算倍率并应用到伤害/治疗公式。法术残页掉落通过施法命中回调中对目标执行 `lootdropper:SpawnLootPrefab()` 实现。以上方案均不依赖引擎修改，完全在 DST Mod API 范围内可行。

**法术升级材料掉落（法术残页）**

释放攻击/治疗类法术命中目标后，**目标**有概率掉落下一等级法术的升级材料（法术残页）。治疗法术的残页由**被治疗目标**掉落。

| 当前法术等级 | 掉落物 | 掉落概率 | 用途 |
|--------------|--------|----------|------|
| 初级 | 中级法术残页 | 15% | 在书架升级为中级 |
| 中级 | 高级法术残页 | 10% | 在书架升级为高级 |
| 高级 | — | — | 已满级 |

- 残页互不通用（如"神锋无影·中级残页"只能用于升级神锋无影）
- 可堆叠（上限 10），可交易
- Prefab：`hw_fragment_<spell>_intermediate`、`hw_fragment_<spell>_advanced`
- 实现要点：施法命中回调中按概率对目标执行 `lootdropper:SpawnLootPrefab(fragment_prefab)`，治疗法术则对被治疗者执行

> ⚠️ 下方 4.C.1 – 4.C.6 各法术目前仅列出初级数值。攻击 / 治疗类（神锋无影、火焰风暴、愈合如初）的中级 / 高级数值待后续细化。火焰熊熊、阿瓦达索命、魂魄出窍无等级区分。

## 6. 研发拆分（给 Coding AI）

### 6.1 里程碑

| 里程碑 | 内容 | 预计工作量 |
|--------|------|------------|
| M1 — 基础框架 | modmain 框架、hw_mana 组件、魔法工作台、制作页签 | 3~5 天 |

| M6 — 联机测试与打磨 | 联机同步调优、PVP 平衡、性能优化、Bug 修复 | 5~7 天 |

### 6.2 风险点

1. **联机同步延迟** — 法术投射物与碰撞判定在高延迟下可能不准确 → 服务端做宽松判定 + 客户端插值
2. **Buff 叠加冲突** — 多个 Buff 同时生效可能产生意外交互 → `hw_buff_manager` 需实现优先级与互斥规则
3. **Mod 兼容性** — 其他添加 `RECIPETABS` 的模组可能冲突 → 使用独特命名空间
4. **性能** — 火焰风暴 AOE 检测大量实体 → 使用空间分区、限制最大命中数

### 6.3 验收标准总则

- 所有功能在 **DST 专用服务器 + 2 客户端** 环境下测试通过
- 所有法术在 **PVP 开/关** 两种模式下行为正确
- 无 crash、无明显卡顿（60 FPS 场景下无掉帧）
- 所有制作配方可在服务器配置中调整
